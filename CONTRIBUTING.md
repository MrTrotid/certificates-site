# Contributing to Certificates Website

Thank you for your interest in contributing! This document outlines how to report bugs and suggest UI/UX improvements.

---

## How to Contribute

### 1. Reporting Bugs

If you find a bug, please create an issue with the following details:

**Bug Report Template:**
```markdown
## Bug Description
[Clear and concise description of the bug]

## Steps to Reproduce
1. Go to '...'
2. Click on '...'
3. See error

## Expected Behavior
[What should happen]

## Actual Behavior
[What actually happens]

## Screenshots
[If applicable, add screenshots]

## Environment
- Browser: [e.g., Chrome, Firefox]
- OS: [e.g., Windows, macOS]
```

### 2. UI/UX Suggestions

Have ideas to improve the design or user experience? We'd love to hear them!

**Suggestion Template:**
```markdown
## UI/UX Suggestion
[Clear description of your suggestion]

## Why This Would Help
[Explain the benefit of this change]

## Mockup/Reference
[If you have a reference design or mockup, include it]
```

## Pull Requests

If you'd like to fix an issue yourself:

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test locally with `npm run dev`
5. Build with `npm run build`
6. Commit with clear messages
7. Push to your fork
8. Create a Pull Request

### PR Guidelines
- Follow existing code style
- Keep changes focused and minimal
- Include screenshots for UI changes
- Test on multiple screen sizes

---

## Code of Conduct

- Be respectful and constructive
- Focus on the issue, not the person
- Welcome new contributors
- Help others learn and grow
